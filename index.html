Panduan Implementasi Mobile

Arsitektur "Tree" untuk Pengisian Data Dashboard Dinamis

Langkah 1

Inisialisasi Filter Tanggal (Draft Awal)

Ambil batas awal bulan untuk mengisi list pilihan bulan (month picker) di aplikasi mobile.

GET /api/dashboards/:ownerId/draft-start

Request:

curl -X GET https://api.desapedia.id/api/dashboards/desa1/draft-start

Sample Response:

{
  "success": true,
  "data": {
    "draftStartMonth": "2025-01"
  }
}

*Mobile harus meng-generate list bulan mulai dari Jan 2025 s/d Bulan Saat Ini.

Langkah 2

Pilih Kategori (List Panels)

Setelah user memilih bulan, tampilkan daftar Panel (Kategori) yang ada di dashboard desa tersebut.

GET /api/dashboards/:ownerId/panels

Sample Response:

{
  "success": true,
  "data": {
    "dashboardName": "Dashboard Desa Maju",
    "panels": [
      { "id": "panel-pop", "title": "Kependudukan", "componentCount": 3 },
      { "id": "panel-health", "title": "Kesehatan", "componentCount": 2 }
    ]
  }
}
Langkah 3

Pilih Isian (List Components & Schema)

Setelah panel dipilih, tampilkan daftar komponen di dalamnya. Endpoint ini mengembalikan formSchema yang digunakan untuk merender UI inputan secara dinamis.

PENTING Gunakan fieldKey sebagai kunci utama saat menyimpan data ke values.

GET /api/dashboards/:ownerId/panels/:panelId/components

Sample Response:

{
  "success": true,
  "data": {
    "panelTitle": "Kependudukan",
    "components": [
      {
        "id": "comp-1",
        "title": "Data Gender",
        "formId": {
          "_id": "form-123",
          "formSchema": {
            "fields": [
              { "fieldKey": "jml_pria", "label": "Jumlah Pria", "type": "number" },
              { "fieldKey": "jml_wanita", "label": "Jumlah Wanita", "type": "number" }
            ]
          }
        }
      }
    ]
  }
}
Langkah 4

Manajemen Data (CRUD)

Gunakan ID Form dari Langkah 3 untuk mengelola data. Pastikan menyertakan filter tanggal agar data tidak bercampur antar bulan.

BACA DATA

GET /api/data/:formId?startDate=...&endDate=...

SIMPAN DATA

POST /api/data
{
  "ownerId": "desa1",
  "formId": "form-123",
  "values": { 
    "jml_pria": 100, 
    "jml_wanita": 105 
  },
  "submittedAt": "2026-03-01T00:00:00Z"
}
